# Matriz de 21st.dev para RJL

## Regla madre

Usar 21st.dev como base de app adaptada al lenguaje RJL. No usarlo como biblioteca de landing pages llamativas.

## Aceptar un componente solo si

- reduce friccion real
- mejora lectura de datos o estado
- encaja con un producto juridico sobrio

## Rechazar un componente si

- impresiona mas de lo que ayuda
- mete animacion o ruido visual innecesario
- hace que cliente, asesor o admin parezcan marketing

## Matriz por categoria

| Categoria 21st.dev | Publico | Cliente | Asesor | Admin | Restricciones |
|---|---|---|---|---|---|
| Navbar | Si | Si | Si | Si | Sobrio, sin marketing ni links promocionales |
| Tabs / Segmented nav | Si | Si | Si | Si | Usarlos para role tabs y subnavs |
| Hero | Si | No como patron principal | No | No | Solo para publico, corto e institucional |
| AI Chat Components | Si | Si | No como patron principal | No | Chat es modulo, no decoracion |
| Input / Textarea | Si | Si | Si | Si | Mantener look juridico y premium |
| Cards | Si | Si | Si | Si | Base de bloques y secciones |
| Stat cards | No protagonistas | Si secundarias | Si | Si | Nada de counters marketineros |
| Data lists / Tables | No | Si en detalle | Si | Si | Densidad operativa para asesor y admin |
| Alerts / Pills / Banners | Si | Si | Si | Si | Ideal para estados, disclaimers y prioridades |
| Forms | Minimo | Si | Si | Si | Cliente para expediente, admin para config |
| Dialogs / Drawers | Si | Si | Si | Si | Preferidos para detalle y edicion puntual |
| Empty states | Si | Si | Si | Si | Deben dar siguiente accion clara |
| Feature Sections | No | No | No | No | Marketing, evitar |
| Testimonials | No | No | No | No | No aportan a producto operativo |
| Pricing Sections | Solo detalle secundario | No | No | No | Nunca dominar la experiencia |
| CTA Sections | No | No | No | No | Evitar bloques de venta puros |
| Logo clouds | No | No | No | No | No aplica al producto |
| Animated text / shaders | No | No | No | No | Rompen sobriedad juridica |

## Seleccion recomendada por experiencia

### Publico

- Navbar
- role tabs
- hero corto
- disclaimer pill
- contenedor de chat
- input primario

### Cliente

- cards de expediente
- grid de celdas para datos laborales
- mosaico de evidencias
- lista breve de consultas
- bloque de chat
- drawers o modales para detalle

### Asesor

- bandeja tipo data list
- lista de clientes seleccionable
- badges de prioridad
- cards compactas de proyeccion
- panel de IA
- CTA fuerte para PDF

### Admin

- KPI cards
- listas o tablas compactas
- drawers de edicion
- filtros
- formularios de configuracion por dominio
- layouts de backoffice
